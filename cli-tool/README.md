# AI Music Tagger - Client

TypeScript CLI client for analyzing audio files and generating comprehensive music metadata.

## Quick Start

### Prerequisites

- Node.js (v18 or higher)
- Backend server running (see [Backend README](../backend/README.md))

### Installation

1. Install dependencies:

```bash
npm install
```

2. Build the TypeScript code:

```bash
npm run build
```

### Usage

**Basic usage:**

```bash
npm start -- --audio "public/audio/Perfekte Welle (SARIAN Remix).mp3"
```

**With custom prompt:**

```bash
npm start -- --audio "public/audio/Perfekte Welle (SARIAN Remix).mp3" --prompt "Limit the genre to one of these: Techno, Minimal Techno, Acid Techno, Industrial Techno, Deep Techno, Hard Techno, Tech House, Progressive Techno"
```

**Update ID3 tags from analysis (analyzes first if needed):**

```bash
npm start -- --audio "public/audio/Perfekte Welle (SARIAN Remix).mp3" --update-metadata
```

**Update ID3 tags with custom merge strategy:**

```bash
npm start -- --audio "public/audio/Perfekte Welle (SARIAN Remix).mp3" --update-metadata --mergeStrategy overwrite
```

**Update ID3 tags with only summary in comment field (override default):**

```bash
npm start -- --audio "public/audio/Perfekte Welle (SARIAN Remix).mp3" --update-metadata --commentStrategy summary
```

**Update ID3 tags with hashtags in comment field:**

```bash
npm start -- --audio "public/audio/Perfekte Welle (SARIAN Remix).mp3" --update-metadata --commentStrategy hashtags
```

**Update ID3 tags with only tags (no summary) in comment field:**

```bash
npm start -- --audio "public/audio/Perfekte Welle (SARIAN Remix).mp3" --update-metadata --commentStrategy tags
```

**Process all audio files in a directory (recursive):**

```bash
npm start -- --audio "public/audio/"
```

**Update metadata for all files in a directory:**

```bash
npm start -- --audio "public/audio/" --update-metadata
```

**Merge all analysis JSON files from a folder:**

```bash
npm run merge-analysis-results -- --folder "public/audio"
```

## Command Line Options

### Main Client (`npm start` / `npm run cli`)

- `--audio` / `-a` (required) - Path to the audio file or directory to analyze. If a directory is provided, all audio files in that directory and subdirectories will be processed recursively.
- `--prompt` / `-p` (optional) - Custom prompt for the analysis
- `--update-metadata` / `-u` (optional) - Update ID3 tags from analysis file. If analysis doesn't exist, it will be created first
- `--mergeStrategy` / `-m` (optional) - Merge strategy for combining existing and new metadata values. Options:
  - `keep-existing` (default) - Only overwrite empty fields, preserve existing non-empty values
  - `overwrite` - Replace existing values with new ones when there's a conflict (same field name)
  - `combine` - Merge arrays, extend CSV values, keep single string values
- `--commentStrategy` / `-c` (optional) - Strategy for formatting the comment field in ID3 metadata. Options:
  - `tags+summary` (default) - Prints tags as CSV string, followed by a line break and the summary
  - `summary` - Uses the summary text from the analysis result
  - `hashtags` - Combines descriptive tags, moods, emotions, and influences into hashtags (e.g., `#Drum&Bass #Energetic #Excitement`)
  - `tags` - Prints tags as CSV string only (no summary)
- `--help` - Show help message

### Merge Analysis Results (`npm run merge-analysis-results`)

- `--folder` / `-f` (required) - Path to the folder containing `*-analysis.json` files to merge
- `--help` - Show help message

## Available Scripts

- `npm run build` - Compile TypeScript to JavaScript in `dist/` directory

## How It Works

### Single File vs Directory Processing

The `--audio` option accepts both individual file paths and directory paths:

- **Single file**: Processes the specified audio file
- **Directory**: Recursively finds all audio files (mp3, flac, wav, m4a, aac, ogg, opus, wma, aiff, aif, m4p, mp4, 3gp) in the directory and subdirectories, then processes each file sequentially

When processing a directory, the tool will:

- Display progress for each file (e.g., "[3/15] Processing: song.mp3")
- Continue processing even if individual files fail
- Show a summary at the end with total files found, successfully processed, and failed files

### Analysis Workflow

1. The client reads the audio file from the specified path
2. Sends the audio file to the backend API at `http://localhost:3000/analyze`
3. Receives comprehensive metadata analysis
4. Saves the analysis as a JSON file next to the original audio file

The analysis file will be named `[original-filename]-analysis.json` and saved in the same directory as the audio file.

### Metadata Update Workflow

When using `--update-metadata`:

1. Checks if an analysis file exists for the audio file
2. If not found, runs the analysis workflow first (see above)
3. Reads the current ID3 tags from the audio file
4. Transforms the analysis result into ID3-compatible metadata
5. Merges existing and new metadata according to the selected merge strategy (see below)
6. Writes the updated metadata to the audio file

### Merge Strategies

The `--mergeStrategy` flag controls how existing and new metadata values are combined:

- **`keep-existing`** (default): Only overwrites empty fields, preserving all existing non-empty values.

- **`overwrite`**: Replaces existing values with new ones when there's a conflict (same field name). Existing fields that don't appear in the new metadata are preserved.

- **`combine`**:
  - **Arrays**: Merges arrays by combining unique values from both existing and new
  - **TXXX entries**:
    - If same description exists and value is CSV/array: Extends existing items with new items
    - If same description exists and value is single string: Keeps existing value
    - If no entry exists: Adds new entry
  - **Single string fields**: Keeps existing value if already present
  - **Encoder fields**: Always updated
  - **Date fields**: Keeps existing value

**Examples:**

```bash
# Use default strategy (keep-existing)
npm start -- --audio "song.mp3" --update-metadata

# Overwrite conflicting fields
npm start -- --audio "song.mp3" --update-metadata --mergeStrategy overwrite

# Combine arrays and extend CSV values
npm start -- --audio "song.mp3" --update-metadata --mergeStrategy combine
```

### Comment Strategies

The `--commentStrategy` flag controls what content is written into the ID3 "comment" metadata field:

- **`tags+summary`** (default): Prints all tags as a comma-separated CSV string, followed by a line break, then the summary text.

  - Format: `tag1, tag2, tag3\nSummary text here`
  - If no tags are available, falls back to the summary only

- **`summary`**: Writes the summary text from the analysis result into the comment field.

- **`hashtags`**: Combines all tags from `descriptiveTags`, `moods`, `emotions`, and `influences` into a single set, then formats them as hashtags (e.g., `#Drum&Bass #Energetic #Excitement #Pendulum`).

  - Whitespaces in tags are replaced with dashes (e.g., "High Energy" → "#High-Energy")
  - Whitespaces around special characters are removed (e.g., "Drum & Bass" → "#Drum&Bass")
  - If no tags are available, falls back to the summary

- **`tags`**: Prints all tags as a comma-separated CSV string only, without the summary.

  - Format: `tag1, tag2, tag3`
  - If no tags are available, falls back to the summary

**Examples:**

```bash
# Use default comment strategy (tags+summary)
npm start -- --audio "song.mp3" --update-metadata

# Use only summary in comment field
npm start -- --audio "song.mp3" --update-metadata --commentStrategy summary

# Use hashtags in comment field
npm start -- --audio "song.mp3" --update-metadata --commentStrategy hashtags

# Use only tags (no summary) in comment field
npm start -- --audio "song.mp3" --update-metadata --commentStrategy tags

# Combine with merge strategy
npm start -- --audio "song.mp3" --update-metadata --mergeStrategy overwrite --commentStrategy hashtags
```

### Merge Analysis Results Workflow

The `merge-analysis-results` script collects all analysis JSON files from a specified folder and merges them into a single structured file.

**Usage:**

```bash
npm run merge-analysis-results -- --folder "public/audio"
```

**What it does:**

1. Scans the specified folder for all files matching the pattern `*-analysis.json`
2. Reads and parses each analysis JSON file
3. Creates a merged JSON file named `merged-analyses.json` in the same folder
4. Logs the name of each file being processed, the total count, and the output file path

**Output structure:**
The generated `merged-analyses.json` file has the following structure:

```json
{
  "createdAt": "2025-01-XX...",
  "analyses": [
    {
      "file": "song1-analysis.json",
      "content": { ... }
    },
    {
      "file": "song2-analysis.json",
      "content": { ... }
    }
  ]
}
```

**Example output:**

```
Found 3 analysis file(s) to process.

Processing: dance-loop-analysis.json
Processing: Perfekte Welle (SARIAN Remix)-analysis.json
Processing: Afterparty-analysis.json

Processed 3 analysis file(s)
Merged analyses written to: /path/to/public/audio/merged-analyses.json
```

## Development

### Building

The project uses TypeScript and must be compiled before running with `npm start`:

```bash
npm run build
```

### Watch Mode

For development, use watch mode to automatically rebuild on changes:

```bash
npm run watch
```

Then in another terminal, run:

```bash
npm start -- --audio "public/audio/Perfekte Welle (SARIAN Remix).mp3"
```

### Direct Execution (No Build)

For quick testing without building, use the CLI script:

```bash
npm run cli -- --audio "public/audio/Perfekte Welle (SARIAN Remix).mp3"
```

## Configuration

The client automatically connects to:

- `http://localhost:3000` in development (default)
- Production URL when `NODE_ENV=production`

Make sure the backend server is running before using the client.

## Troubleshooting

### "Cannot find module 'dist/index.js'"

Run `npm run build` first to compile the TypeScript code.

### "Backend request failed"

Make sure the backend server is running on port 3000. See the [Backend README](../backend/README.md) for setup instructions.

### "File not found" or "Path not found"

Check that the audio file or directory path is correct and exists. If processing a directory, ensure it contains audio files with supported extensions.
